# Agent Documentation Contract

This directory contains the project documentation that every AI agent must follow before changing this repository.

## Required Reading Order

1. `project.md`
2. `frontend.md` when working in `web/`
3. `backend.md` when working in `cmd/`, `internal/`, or `db/`
4. `devops.md` when working on Docker, Compose, Makefile, or runtime commands
5. `verification.md` before finishing any code change

## Compliance Rule

Agents must treat these docs as repository policy. If a user request conflicts with this contract, follow the newest explicit user request and keep the change as small as possible.

## Update Rule

When project conventions change, update the relevant file in this directory in the same change set as the code change.
